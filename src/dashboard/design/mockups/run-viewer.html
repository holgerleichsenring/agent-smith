Our services still run the legacy MassTransit bus together with MediatR for in-process dispatch. Wolverine now covers both roles, so we want to consolidate onto a single messaging stack across the three DigitalAccessPortal services and drop the two old dependencies.

This should be behaviour-preserving — every existing message flow must keep working exactly as before. No new features, just the migration and the cleanup.

What we need

  • Replace the MassTransit host/bus configuration with a Wolverine host.
  • Move all consumers to Wolverine handlers and swap IMediator usage for the Wolverine message bus.
  • Remove the MassTransit and MediatR package references once nothing depends on them.
  • Keep the integration test suite green.

The target architecture is attached; the flaky-consumer screenshot shows the retry storm we currently hit under load, which the migration should also settle.
